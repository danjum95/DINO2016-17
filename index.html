Opracowanie komend git

Zadania dla grupy 1CE

  • git config --global user.email "" - ustawia globalnie adres e-mail użytkownika używany do commitowania
  • git commit -a -m "" - automatycznie stage'uje wszystkie zmodyfikowane i usunięte pliki (za wyjątkiem nowych) i dodaje commit z komentarzem ""
  • git diff --staged - pokazuje różnice pomiędzy ostatnim commitem a zestage'owanymi zmianami
  • git checkout -b "branch" - przełącza do gałęzi o nazwie "branch". Jeżeli nie istnieje gałąź o takiej nazwie, tworzy ją.
  • git log --graph --all - wyświetla graficzną (choć tekstową) reprezentację historii wszystkich commitów w repozytorium
    Uwaga: Opcja --graph nie może być łączona z opcja --no-walk (która pokazuje tylko commity, ale bez ich poprzedników)
  • git branch -d "branch" - usuwa gałąź o nazwie "branch"
  • git remote add "Grażyna" "git url" - dodaje zdalne repozytorium pod nazwą "Grażyna"
  • git fetch "remote" "branch" - pobiera zmiany ze zdalnego repozytorium o nazwie "remote", z gałęzi "branch". W przeciwieństwie do pulla nie merguje lokalnego repozytorium z głównym.

Zadanie dla grupy 1CC

  • git config --global user.name "" - konfiguracja własnej nazwy użytkownika. Opcja '--global' sprawia że konfiguracja zostanie dokonana jednorazowo
  • git add . - komenda dokonująca update w Index używajac aktualnego kontentu znalezionego w 'working tree', żeby przygotować 'content staged' na nastepny commit. Stages new and modified, without deleted.
  • git reset HEAD - resetuje wszystkie spisy 'Index' w HEAD, pozostawiając 'working tree' bez zmian i raportuje to, co zostało zmienione. Dodatkowo można podać mode, domyślny to --mixed
  • git checkout "branch" - przełącza na gałąź branch - po jej wykonaniu HEAD wskazuje na branch
  • gitk --all - otwiera program ze środowiskiem graficznym, który umożliwia podejrzenie wszystkich wprowadzanych zmian na repozytorium
  • git branch --merged - wypisuje wszystkie gałęzie połączone z HEAD (głowica gałęzi na której się znajdujemy)
  • git branch "branch" "commit hash" - tworzy nową gałąź 'branch' a jako jej head (głowicę) ustawia 'commit hash'
  • git pull "remote" "branch" - Pobiera gałąź 'branch' ze zdalnego repozytorium 'remote' i zapisuje ją w lokalnej gałęzi 'branch'

Zadania dla grupy 1CF

  • git config --global color.ui true - Komunikaty generowane przez komendy git'a w terminalu będą miały różne kolory.
  • git commit "NazwaPliku" - Tworzy commit składający się z plików podanych jako argument "NazwaPliku". Pozostałe pliki w stanie staged nie są dodane do tego commita.
  • git branch "name" - Tworzy nową gałąź o nazwie "name".
  • git branch -v - Wypisuje nazwę aktywnego brancha wraz z szczegółami ostatniego commita (początek skrótu sha1 oraz temat).
  • git push "remote" "branch" - Wysyła zmiany z ostatniego commita w lokalnej gałęzi "branch" do zdalnego repozytorium "remote".
  • git log "branch1" --not "branch2" - Wyświetli historię commitów "branch1" z pominięciem commitów będących częścią "branch2".
  • git blame -C "filename" - Wyszukuje fragmenty kodu przekopiowane z innych plików będących częścią tego samego commita.
  • git bisect (start, bad, good, reset) - Pozwala na binarne wyszukiwanie commitu w którym pojawiła się konkretna zmiana.
    • git bisect start - rozpoczyna wyszukiwanie
    • git bisect bad "name" - znakuje commit o nazwie "name" w którym pojawiła się zmiania
    • git bisect good "name" - znakuje commit o nazwie "name" w którym nie ma szukanej zmiany
    • git bisect reset - przywraca stan do ostatniego aktywnego commitu przed git bisect start
